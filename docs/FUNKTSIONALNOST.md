# Функционал сервиса 1pay_site_mon

Сервис предназначен для **мониторинга веб-страниц и проверки контента**: загрузка URL, извлечение текста, опциональные скриншоты, обход сайта и поиск запрещённых слов/фраз в тексте. Результаты доступны через веб-интерфейс и API.

---

## 1. Назначение сервиса

- **Загрузка страниц** — по списку URL или путём обхода сайта (краулинг).
- **Извлечение текста** — основной контент из HTML (Mozilla Readability), нормализованный текст.
- **Скриншоты** — опционально по каждой странице (Playwright, viewport или полная страница).
- **Проверка запрещённых терминов** — поиск заданных слов/фраз в извлечённом тексте с разными режимами сопоставления.
- **Результаты** — веб-интерфейс и API: статус задачи, постраничные результаты, отмена, экспорт в CSV.

Типичное применение: контроль сайтов на соответствие требованиям (например, отсутствие запрещённых формулировок).

---

## 2. Режимы работы

### 2.1 Режим «Список» (list)

- Пользователь задаёт **список URL** (вставка текста или загрузка CSV).
- Каждый URL обрабатывается один раз: загрузка → извлечение текста → опционально скриншот → опционально проверка запрещённых слов.
- Подходит для проверки заранее известных страниц.

### 2.2 Режим «Обход» (crawl)

- Задаётся **стартовый URL** (seed).
- Сервис извлекает ссылки из HTML и обходит сайт в ширину (BFS) с ограничениями:
  - **maxPages** — максимум страниц (по умолчанию 100),
  - **maxDepth** — максимальная глубина от стартовой страницы (по умолчанию 3),
  - **sameHostOnly** — только ссылки с того же хоста (по умолчанию да),
  - **includePatterns** / **excludePatterns** — регулярные выражения для включения/исключения URL.
- Подрежимы обхода: только отправленные URL, только стартовая страница или полный краулинг.

---

## 3. Обработка страниц

### 3.1 Загрузка HTML

- HTTP-запросы через **undici** с таймаутом и лимитом размера ответа.
- **SSRF-защита**: перед загрузкой проверяется хост — блокируются приватные IP и запрещённые имена (localhost и т.п.).
- Настраиваются: таймаут (по умолчанию 25 с), максимум редиректов (10), лимит размера ответа (10 МБ), повторные попытки (2).

### 3.2 Извлечение текста

- HTML парсится (**JSDOM**), основной контент выделяется через **Mozilla Readability**.
- При неудаче Readability используется очищенный текст из `body`.
- Результат — нормализованный plain text (пробелы, переносы); опционально обрезка по лимиту символов (по умолчанию 300 000).

### 3.3 Скриншоты (опционально)

- **Playwright** (Chromium): viewport (1280×720 по умолчанию) или полная страница (склейка скролла через Sharp).
- Файлы сохраняются в `server/storage/screenshots/`, отдаются по `/static/screenshots/`.
- Параллельность скриншотов ограничена (по умолчанию 3), таймаут навигации (35 с).

---

## 4. Проверка запрещённых слов

Пользователь задаёт список терминов; сервис ищет их в извлечённом тексте страницы.

### 4.1 Режимы сопоставления (matchMode)

| Режим | Описание |
|-------|----------|
| **exact_substring** | Точное вхождение подстроки в текст |
| **word** | Совпадение целого слова (границы слова) |
| **smart_stem** | Стемминг (RU/EN, Snowball), совпадение по основе слова |
| **smart_fuzzy** | Нечёткое совпадение (расстояние Дамерау–Левенштейна) |

### 4.2 Дополнительные настройки

- **phraseMode** — поиск фразы как последовательности токенов/стемов (`token_stem_sequence` или отключено).
- **languageMode** — язык для стемминга: авто, русский, английский.
- **caseSensitive** — учёт регистра.
- **minWordLength** — минимальная длина слова для учёта.
- **maxMatchesPerTermPerPage** — максимум совпадений одного термина на страницу.
- **fuzzy** — для smart_fuzzy: максимальное расстояние, максимум проверяемых токенов.

### 4.3 Результат проверки

- Количество совпадений по каждому термину.
- Тип совпадения (exact_substring, word, smart_stem, smart_fuzzy, phrase).
- Сниппеты с контекстом вхождения.
- Итоговая сводка по нарушениям (violations) по задаче.

---

## 5. Задачи (jobs)

### 5.1 Жизненный цикл

1. **Создание** — POST с параметрами (режим, URL/seed, опции). Ответ: `202` и `jobId`.
2. **Выполнение** — асинхронная обработка в фоне, прогресс и результаты пишутся в хранилище задачи.
3. **Статусы**: `queued` → `running` → `done` / `cancelled` / `failed`.

### 5.2 Прогресс и сводка

- **discovered** — найдено страниц (в режиме crawl).
- **queued** — в очереди на обработку.
- **processed** — успешно обработано.
- **failed** — ошибки загрузки/обработки.
- **violations** — страниц с найденными запрещёнными терминами.

### 5.3 Управление

- **Отмена** — POST `/api/jobs/:jobId/cancel` останавливает выполнение (AbortController).
- Хранилище задач — **in-memory** (без персистентности между перезапусками).

---

## 6. API

| Метод | Путь | Назначение |
|-------|------|------------|
| POST | `/api/jobs` | Создать задачу. Тело: `{ mode, urls?, seedUrl?, options? }`. Ответ: `202 { jobId }`. |
| GET | `/api/jobs/:jobId` | Метаданные и прогресс задачи: jobId, mode, createdAt, status, progress, summary. |
| GET | `/api/jobs/:jobId/results` | Постраничные результаты. Параметры: `cursor`, `limit` (макс. 100). Ответ: `{ items, nextCursor }`. |
| POST | `/api/jobs/:jobId/cancel` | Отменить задачу. Ответ: `{ ok: true }`. |

Статика: `GET /static/screenshots/*` — раздача файлов скриншотов.

---

## 7. Ввод данных

### 7.1 URL

- **Список**: вставка текста (по одному URL на строку) или загрузка CSV.
- CSV: автоопределение разделителя, колонка с URL по имени (`url`, `link`, `href` и т.п.).
- В режиме crawl — один стартовый URL (seed).

### 7.2 Запрещённые термины

- Вставка текста (по одному термину/фразе на строку) или загрузка CSV.
- Парсинг и нормализация на клиенте и при необходимости на сервере.

---

## 8. Результаты и экспорт

- **Таблица результатов**: URL, заголовок, статус, длина текста, наличие скриншота, нарушения.
- **Фильтры**: все / успешные / с ошибками, только с нарушениями, поиск по тексту.
- **Детальный просмотр**: метаданные, скриншот, совпадения запрещённых терминов, скачивание текста.
- **Экспорт CSV**: полный отчёт по результатам и отдельно только нарушения (violations).

---

## 10. Технологический стек

- **Сервер**: Node.js (≥18), Fastify, CORS, статика; undici, JSDOM, @mozilla/readability; Playwright, Sharp; стемминг (snowball-stemmers), damerau-levenshtein; TypeScript, Vitest.
- **Клиент**: React 18, Ant Design, Vite, TypeScript.
- **Сборка**: pnpm workspace (client, server); в dev запросы `/api` и `/static` проксируются на сервер.
- **Развёртывание**: переменная `PORT` (по умолчанию 3000); Docker — один образ (клиент + сервер + Playwright Chromium), volume для скриншотов.

---

## 11. Безопасность

- **SSRF**: проверка разрешённости хоста и запрет доступа к приватным IP и локальным адресам перед загрузкой.
- Ограничения на размер ответа и таймауты снижают риски при обработке внешних URL.
